# CLAUDE.md

## プロジェクト概要

Typstを使用したプレゼンテーション作成リポジトリ。

## ディレクトリ構成
```
.
├── CLAUDE.md
├── *.typ             # それぞれのプレゼンテーションファイル
├── theme/            # カスタムテーマ定義
│   └── theme.typ
├── assets/           # 画像・図表
│   └── images/
└── output/           # ビルド成果物（.gitignore対象）
```

## ビルドコマンド

仮にプレゼンテーションファイルを `foobar.typ` とした場合:

- プレビュー: `typst watch foobar.typ output/foobar.pdf`
- ビルド: `typst compile foobar.typ output/foobar.pdf`

## Typst コーディング規約

- スライド区切りには必ず見出し（`= タイトル`）またはテーマの改ページ関数を使う
- 日本語フォントは `#set text(font: "Noto Sans CJK JP")` で統一
- 色定義は `theme/theme.typ` に集約し、本文中でハードコードしない
- 画像は `assets/images/` 配下に配置し、相対パスで参照する

## スライドテーマ

- Polyluxを使用: `#import "@preview/polylux:0.4.0": *`
  （または使用するテーマパッケージに応じて変更）
- テーマ設定は `theme/theme.typ` で定義

## コンテンツ作成ルール

- 1スライドにつき1つの主張（情報を詰め込みすぎない）
- 箇条書きは最大5項目まで
- コードブロックには必ず言語指定をつける
- 図表にはキャプションを付与する

## 言語

- スライド本文: 日本語または英語
- コメント・変数名: 英語可

## 必ずやること

- rootディレクトリ直下の1つの.typファイルに全スライドを詰め込む。プレゼンテーションごとにファイルは分ける。

## やってはいけないこと

- output/ 配下のファイルをGit管理に含めない
- .typファイルを分割することはしない。
- テーマカラーを直接 `rgb()` でスライド本文に書かない
